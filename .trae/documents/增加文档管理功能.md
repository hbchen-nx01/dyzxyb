# 增加文档管理功能实施计划

## 一、功能概述
在现有班组管理系统中增加"文档管理"功能模块，用于上传、存储和管理各类文档资源。

## 二、实施步骤

### 1. 数据模型设计与实现
- 在`models.py`中添加`Document`和`DocumentCategory`模型
- 实现文档的分类管理、上传时间、文件名、文件路径等核心字段

### 2. 路由蓝图开发
- 创建`routes/document_management.py`文件
- 实现文档管理的核心功能路由：
  - 文档列表展示与查询
  - 文档上传功能
  - 文档编辑功能
  - 文档删除功能
  - 文档下载功能

### 3. 模板文件创建
- 创建`templates/document_management/`目录
- 实现以下模板文件：
  - `index.html` - 文档列表页面
  - `add.html` - 文档上传页面
  - `edit.html` - 文档编辑页面

### 4. 首页入口添加
- 在`index.html`中添加文档管理模块的入口卡片
- 将其放置在所有现有模块的最后

### 5. 路由注册
- 在`app.py`中导入并注册文档管理路由蓝图

## 三、技术要点

### 数据模型
- `DocumentCategory`：文档分类表，支持多级分类
- `Document`：文档表，包含分类、文件名、文件路径、上传人、上传时间等字段

### 功能特性
- 支持文档分类管理
- 支持多种文档格式上传
- 支持文档查询和筛选
- 支持文档下载和预览
- 支持文档信息编辑
- 支持文档删除

## 四、文件结构

```
├── models.py                 # 添加文档管理数据模型
├── routes/
│   └── document_management.py  # 文档管理路由
├── templates/
│   └── document_management/  # 文档管理模板
│       ├── index.html
│       ├── add.html
│       └── edit.html
└── app.py                    # 注册文档管理路由
```

## 五、实现细节

1. **数据模型**：
   - `DocumentCategory`模型包含id、name、parent_id、description等字段
   - `Document`模型包含id、title、category_id、file_path、file_name、file_size、uploader、upload_date等字段

2. **路由设计**：
   - `/document-management/` - 文档列表
   - `/document-management/add` - 上传文档
   - `/document-management/edit/<id>` - 编辑文档
   - `/document-management/delete/<id>` - 删除文档
   - `/document-management/download/<id>` - 下载文档

3. **模板设计**：
   - 采用与现有模块一致的设计风格
   - 实现文档列表、上传表单、编辑表单等界面

4. **首页入口**：
   - 添加文档管理卡片，图标使用`bi bi-file-earmark`
   - 放置在现有模块的最后位置

## 六、预期效果

- 在首页模块列表的最后增加"文档管理"入口
- 点击入口可进入文档管理页面
- 支持文档的上传、编辑、删除和下载功能
- 支持按分类和关键词查询文档
- 界面风格与现有系统保持一致
